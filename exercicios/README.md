## Política de prazos

- Entregas até 23:59 do prazo (horário Brasília).
- Atrasos: -10%/dia, máx 3 dias. Após isso, zero.
- 1 (uma) prorrogação justificada por aluno/grupo, mediante e-mail antes do prazo.

## Formato de entrega

- Repositório GitHub público (ou privado convidando o prof) com README de execução completo.
- Vídeo demo quando solicitado (Loom ou YouTube unlisted).
- Relatórios em PDF ou markdown (formato ABNT ou IEEE).

## Critério eliminatório

**Projeto Final em Grupo** é obrigatório. Grupo que não apresentar não passa.

## Projeto Final — temas disponíveis

Grupos de 3-4 alunos escolhem um tema:

1. Automação Mobile End-to-End
2. Native UI Testing (Espresso + XCUITest com Page Objects)
3. Performance Mobile Testing
4. Mobile Security Testing (OWASP MASVS + MobSF)
5. Visual AI em Mobile
6. Test Generation com LLM (Claude API + Maestro)
7. AI Agent para Exploratory Mobile (AppAgent ou DroidBot+LLM)
8. CI/CD Pipeline Mobile (GitHub Actions + Fastlane)
9. Mobile API Contract Testing (Pact)
10. Accessibility Testing Mobile
